)
staticprivate

Check for cover violation if Varnode is implied.

Marking a Varnode as implied causes the input Varnodes to its defining op to propagate farther in the output. This may cause eventual variables to hold different values at the same point in the code. Any input must test that its propagated Cover doesn't intersect its current Cover.

Parameters
datais the function being analyzed
vnis the given Varnode
Returns
true if there is a Cover violation
